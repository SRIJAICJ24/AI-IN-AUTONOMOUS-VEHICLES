2.3 Creation of Database and Tables - DDL Commands

Note:
In the current project, the SQLite database file is `drivelearn.db`. The database and tables are created by the application using the following DDL commands.

```sql
PRAGMA foreign_keys = ON;

CREATE TABLE IF NOT EXISTS AI_Models (
    model_id INTEGER PRIMARY KEY AUTOINCREMENT,
    version_name TEXT NOT NULL UNIQUE,
    architecture TEXT NOT NULL,
    accuracy_score REAL NOT NULL CHECK (accuracy_score BETWEEN 0.0 AND 1.0)
);

CREATE TABLE IF NOT EXISTS Scenarios (
    scenario_id INTEGER PRIMARY KEY AUTOINCREMENT,
    weather_type TEXT NOT NULL CHECK (weather_type IN ('Rain', 'Fog', 'Sunny')),
    light_level TEXT NOT NULL CHECK (light_level IN ('Day', 'Night')),
    road_type TEXT NOT NULL CHECK (road_type IN ('Highway', 'Urban')),
    difficulty TEXT NOT NULL CHECK (difficulty IN ('Low', 'Medium', 'High', 'Critical'))
);

CREATE TABLE IF NOT EXISTS Vehicles (
    vehicle_id INTEGER PRIMARY KEY AUTOINCREMENT,
    make_model TEXT NOT NULL,
    ai_model_id INTEGER NOT NULL,
    status TEXT NOT NULL,
    FOREIGN KEY (ai_model_id) REFERENCES AI_Models(model_id) ON DELETE RESTRICT
);

CREATE TABLE IF NOT EXISTS Test_Runs (
    run_id INTEGER PRIMARY KEY AUTOINCREMENT,
    vehicle_id INTEGER NOT NULL,
    scenario_id INTEGER NOT NULL,
    test_date TEXT NOT NULL,
    avg_speed REAL NOT NULL DEFAULT 0,
    FOREIGN KEY (vehicle_id) REFERENCES Vehicles(vehicle_id) ON DELETE RESTRICT,
    FOREIGN KEY (scenario_id) REFERENCES Scenarios(scenario_id) ON DELETE RESTRICT
);

CREATE TABLE IF NOT EXISTS Frames (
    frame_id INTEGER PRIMARY KEY AUTOINCREMENT,
    run_id INTEGER NOT NULL,
    timestamp TEXT NOT NULL,
    image_path TEXT NOT NULL,
    FOREIGN KEY (run_id) REFERENCES Test_Runs(run_id) ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS Detections (
    detection_id INTEGER PRIMARY KEY AUTOINCREMENT,
    frame_id INTEGER NOT NULL,
    object_type TEXT NOT NULL,
    confidence_score NUMERIC NOT NULL CHECK (confidence_score BETWEEN 0.0 AND 0.999),
    FOREIGN KEY (frame_id) REFERENCES Frames(frame_id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_test_runs_scenario_id
ON Test_Runs(scenario_id);

CREATE INDEX IF NOT EXISTS idx_detections_confidence_score
ON Detections(confidence_score);
```

3.2 Queries based on Aggregate Functions

Question 1:
What is the average detection confidence for each AI model in the current project?

SQL Statement:
```sql
SELECT
    m.version_name,
    ROUND(AVG(d.confidence_score), 3) AS avg_confidence,
    COUNT(d.detection_id) AS detections
FROM AI_Models m
JOIN Vehicles v ON v.ai_model_id = m.model_id
JOIN Test_Runs tr ON tr.vehicle_id = v.vehicle_id
JOIN Frames f ON f.run_id = tr.run_id
JOIN Detections d ON d.frame_id = f.frame_id
```

Question 1:
Which AI model has the highest average detection confidence?

SQL Statement:
```sql
SELECT model_name, avg_confidence
FROM (
    SELECT
        m.version_name AS model_name,
        ROUND(AVG(d.confidence_score), 3) AS avg_confidence
    FROM AI_Models m
    JOIN Vehicles v ON v.ai_model_id = m.model_id
    JOIN Test_Runs tr ON tr.vehicle_id = v.vehicle_id
    JOIN Frames f ON f.run_id = tr.run_id
    JOIN Detections d ON d.frame_id = f.frame_id
    GROUP BY m.model_id, m.version_name
)
WHERE avg_confidence = (
    SELECT MAX(avg_confidence)
    FROM (
        SELECT ROUND(AVG(d2.confidence_score), 3) AS avg_confidence
        FROM AI_Models m2

        JOIN Frames f2 ON f2.run_id = tr2.run_id
        JOIN Detections d2 ON d2.frame_id = f2.frame_id
        GROUP BY m2.model_id
    )
);
```

Question 1:
Create a view that summarizes average confidence and detection count by model and scenario.

SQL Statement:

```sql
CREATE VIEW IF NOT EXISTS vw_model_scenario_summary AS
SELECT
    m.version_name,
    s.weather_type || ' ' || s.light_level AS scenario_name,
    ROUND(AVG(d.confidence_score), 3) AS avg_confidence,

JOIN Detections d ON d.frame_id = f.frame_id

SELECT *
FROM vw_model_scenario_summary
```

Create a view to list rain and fog frames where the minimum detection confidence is below `0.60`.

```sql
    tr.run_id,
    v.make_model,
```

3.7 Complex queries based on Triggers

Note:
These trigger examples are written for the current DriveLearn schema. For demonstration, they use temporary log tables and are intended to be executed inside a transaction so the base project data remains unchanged after testing.

Question 1:
How can a trigger log every new test run inserted into `Test_Runs`?

SQL Statement:
```sql
CREATE TEMP TABLE run_insert_log (
    log_id INTEGER PRIMARY KEY AUTOINCREMENT,
    run_id INTEGER,
    vehicle_id INTEGER,
    scenario_id INTEGER,
    avg_speed REAL
);

CREATE TEMP TRIGGER trg_log_run_insert
AFTER INSERT ON Test_Runs
BEGIN
    INSERT INTO run_insert_log (run_id, vehicle_id, scenario_id, avg_speed)
    VALUES (NEW.run_id, NEW.vehicle_id, NEW.scenario_id, NEW.avg_speed);
END;

INSERT INTO Test_Runs (vehicle_id, scenario_id, test_date, avg_speed)
VALUES (1, 2, '2026-03-23T10:00:00', 64.50);

SELECT run_id, vehicle_id, scenario_id, avg_speed
FROM run_insert_log;
```

Output:
```text
run_id | vehicle_id | scenario_id | avg_speed
-------+------------+-------------+----------
21     | 1          | 2           | 64.5
```

Question 2:
How can a trigger record a vehicle status update?

SQL Statement:
```sql
CREATE TEMP TABLE vehicle_status_log (
    vehicle_id INTEGER,
    old_status TEXT,
    new_status TEXT
);

CREATE TEMP TRIGGER trg_log_vehicle_status
AFTER UPDATE OF status ON Vehicles
BEGIN
    INSERT INTO vehicle_status_log (vehicle_id, old_status, new_status)
    VALUES (OLD.vehicle_id, OLD.status, NEW.status);
END;

UPDATE Vehicles
SET status = 'Maintenance'
WHERE vehicle_id = 3;

SELECT vehicle_id, old_status, new_status
FROM vehicle_status_log;
```

Output:
```text
vehicle_id | old_status | new_status
-----------+------------+-----------
3          | Standby    | Maintenance
```

Question 3:
How can a trigger log frame details before a frame is deleted?

SQL Statement:
```sql
CREATE TEMP TABLE frame_delete_log (
    frame_id INTEGER,
    run_id INTEGER,
    image_path TEXT
);

CREATE TEMP TRIGGER trg_log_frame_delete
BEFORE DELETE ON Frames
BEGIN
    INSERT INTO frame_delete_log (frame_id, run_id, image_path)
    VALUES (OLD.frame_id, OLD.run_id, OLD.image_path);
END;

DELETE FROM Frames
WHERE frame_id = 200;

SELECT frame_id, run_id, image_path
FROM frame_delete_log;
```

Output:
```text

---------+--------+-------------------------
200      | 20     | live/run_20/frame_10.jpg
```

3.8 Complex queries based on Cursors

Note:
SQLite does not support stored-procedure style SQL cursors like Oracle or MySQL. In the current DriveLearn project, cursor processing is done in Python through `sqlite3.Cursor` in files such as `app.py` and `query_db.py`. The following examples therefore use the SQL executed by the cursor and the rows returned by that cursor.

Question 1:
Using a cursor, how can we fetch all active vehicles along with their assigned AI model?

SQL Statement:
```sql
SELECT
    v.vehicle_id,
    v.make_model,

WHERE v.status = 'Active'
```

Question 3:
Using a cursor, how can we iterate through scenario statistics for reporting?
